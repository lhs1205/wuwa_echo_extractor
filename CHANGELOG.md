# 명조 에코 스탯 추출기 — 개발 변경 이력

## 현재 버전 기준 주요 기능
- Tesseract.js v4 브라우저 OCR (API 키 불필요)
- 한국어 + 영어 동시 인식 (`kor+eng`)
- 에코 부옵션 5개 추출, 최대 5장 동시 처리
- 퍼지 매칭으로 OCR 노이즈 흡수
- 유효 등급값 스냅 (8등급 테이블 기준)
- 결과 테이블 인라인 편집 + 클립보드 복사

---

## 개발 이력

### 전처리 파이프라인 구조
```
원본 이미지 크롭 (x=73%~100%, y=28%~45%)
  → 2× 확대
  → Laplacian 샤프닝 (획 경계 강조)
  → 노란 박스 억제 (isYellow: r>120, b<100, r-b>80)
  → Otsu 적응형 이진화 (고정 임계값 120 → 자동 계산)
  → dilation 2패스 (whites≥6, 테두리 잔선 제거)
  → Tesseract OCR (PSM 6)
```

---

### OCR 파싱 — 주요 수정 이력

#### 1. 크로스 스크립트 페널티
- **문제**: `방억` (방어력 OCR 노이즈) → `HP` (ASCII 2글자)로 매칭됨
- **원인**: 레벤슈타인 거리가 동일할 때 ASCII 후보가 먼저 선택
- **해결**: 한국어 입력 vs ASCII 후보에 `+ceil(len*0.5)` 페널티 추가

#### 2. 소수점 앞 숫자 누락 복원
- **문제**: OCR이 `?.4%` → `.4%` (앞 자리 드랍)
- **해결**: 값이 `.`으로 시작하면 `0` 앞에 추가, fallback 정규식 추가

#### 3. 노란 유효옵 박스 억제
- **문제**: 유효옵 강조 박스 (황금색) → 이진화 시 검은 사각형으로 변환 → OCR 간섭
- **원인**: 황색 픽셀 휘도 ~190 > 임계값 120 → 검은색 처리
- **해결**: 픽셀 수준 isYellow 감지 (`r>120 && b<100 && r-b>80`) → 흰색으로 강제

#### 4. dilation 테두리 제거
- **문제**: isYellow 억제 후 박스 테두리 잔선(1~2px) OCR 간섭
- **해결**: 이진화 후 dilation 2패스 (8-이웃 중 ≥6개 흰색인 검은 픽셀 → 흰색)
- **주의**: 4패스는 얇은 획("1" 등) 삭제 → 2패스로 제한

#### 5. Laplacian 샤프닝 + Otsu 임계값
- **배경**: "17.4%" → "7.4%"로 읽힘 (1 드랍)
- **원인 A**: 고정 임계값 120이 이미지 밝기 분포에 따라 획을 지우는 경우 존재
- **원인 B**: 4패스 dilation이 얇은 "1" 획 삭제
- **해결**: Laplacian 샤프닝(5, -1 kernel)으로 획 경계 강조 + Otsu 임계값으로 적응형 이진화

#### 6. HP 영문 OCR 오인식 대응
- **문제**: OCR이 `HP` → `10` (H→1, P→0) 또는 `18` (H→1, P→8)로 읽음
- **해결 A**: `lang: 'kor+eng'`으로 영어 모델 추가
- **해결 B**: `DIGIT_OCR_ALIAS = { '10': 'HP', '18': 'HP' }` 직접 매핑
- **해결 C**: rawName 앞쪽 단독 숫자 토큰 제거 ("1 10" → "10")

#### 7. "11" → 한글 "거" 오인식
- **문제**: `11.6%` → OCR이 `거 .6%`로 읽음 ("11"의 획이 "거"와 유사)
- **해결**: `DIGIT_LOOKALIKE_SUFFIX = { '거': '11' }` — 이름 끝에 "거"가 있고 값이 `0.X%`이면 "11"을 앞에 복원

#### 8. 숫자 파싱 정규식 개선
- **문제**: "1 10 8.6%" 파싱 시 `[\d\s]*\d` 패턴이 "10 8"을 하나의 숫자로 묶음
- **해결**: `\d+(?:\s*\.\s*\d+)?`로 변경 — 정수 + 선택적 소수부만 허용

#### 9. OCR 9→0 오인식 교정
- **문제**: `7.9%` → OCR이 `7.0%`로 읽음, nearest-snap으로 `7.1%` 오교정
- **해결**: nearest-snap 전에 각 '0'을 '9'로 치환해 STAT_VALID_VALUES에 정확히 있으면 우선 사용
- **안전성**: 이미 유효값인 경우(예: 방어력 `40`, 공명 효율 `10.0%`) 치환 없이 통과

#### 10. HP flat 필터 오탐 수정
- **문제**: `isBaseStatNoise`가 값 > 150인 flat 스탯을 기본 스탯으로 오분류
- **원인**: HP 부옵션 유효값이 320~580으로 모두 150 초과
- **해결**: 임계값 150 → 1000 (부옵션 최대 HP=580, 기본 스탯은 수천 이상)

### 퍼지 매칭 구조
```
입력 → 공백 제거 → 앞뒤 숫자 노이즈 제거 → DIGIT_OCR_ALIAS 체크
     → Levenshtein 거리 계산 (threshold: max(2, floor(len*0.38)))
     → 크로스 스크립트 페널티 (한국어 vs ASCII)
     → ALIAS_MAP 정규화 (예: "공명 해방 피해 보너스" → "공명 해방 피해")
```

### 유효값 스냅 우선순위
```
1. 이미 유효값 → 그대로
2. 0→9 치환 후 유효값 → 교정 (OCR 오인식)
3. nearest-valid → 스냅 (파란 점 표시)
```
